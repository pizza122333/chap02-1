1. OpenCV를 이용하여 개발된 제품 조사 및 기능 설명

OpenCV는 컴퓨터 비전과 이미지 처리를 위한 오픈소스 라이브러리로 다양한 제품에서 사용된다.

① 얼굴 인식 시스템

예: 스마트폰 얼굴 잠금 해제

기능

* 카메라 영상에서 얼굴 검출
* 사용자 얼굴과 데이터베이스 비교
* 인증 후 잠금 해제

활용 예

* 스마트폰 보안 시스템
* 출입 통제 시스템


② 자율주행 자동차 시스템

예: Tesla의 자율주행 기술

기능

* 도로 인식
* 차선 인식
* 보행자 검출
* 교통 표지판 인식

OpenCV 기술을 이용하여 차량이 주변 환경을 인식할 수 있다.


③ 산업 자동화 검사 시스템

기능

* 제품 불량 검사
* 크기 및 형태 분석
* 색상 검출

공장에서 생산된 제품의 품질 검사를 자동화하는 데 사용된다.


2. 소프트웨어 분야에서 라이브러리의 의미

라이브러리(Library)는 프로그램 개발에 필요한 기능을 미리 만들어 놓은 코드 모음이다.

개발자는 모든 기능을 직접 작성하지 않고 라이브러리를 이용하여 프로그램을 개발할 수 있다.

특징

* 재사용 가능한 코드 제공
* 개발 시간 단축
* 프로그램 유지보수 용이

예시

* 이미지 처리 기능
* 수학 계산 기능
* 네트워크 통신 기능

예를 들어 OpenCV는 이미지 처리 기능을 제공하는 라이브러리이다.


3. 오픈소스의 의미

오픈소스(Open Source)는 소프트웨어의 소스 코드가 공개되어 누구나 자유롭게 사용, 수정, 배포할 수 있는 소프트웨어를 의미한다.

특징

* 소스 코드 공개
* 자유로운 수정 및 재배포 가능
* 개발자 공동 협력 가능

대표적인 오픈소스 예

* Linux
* OpenCV
* Apache HTTP Server

오픈소스는 소프트웨어 기술 발전에 큰 역할을 하고 있다.


4. Windows 환경변수 PATH의 기능

Microsoft Windows의 환경변수 PATH는 실행 가능한 프로그램의 위치를 운영체제가 찾을 수 있도록 경로를 저장하는 변수이다.

기능

* 명령어 실행 시 프로그램 위치 검색
* 프로그램 실행을 편리하게 해줌

예

PATH에 다음 경로가 등록되어 있을 경우

C:\Program Files\OpenCV\bin

사용자는 해당 폴더로 이동하지 않고도 프로그램을 실행할 수 있다.


5. 프로그램 개발 과정 (편집, 컴파일, 링크, 실행)

프로그램 개발은 다음 4단계를 거친다.

① 편집(Edit)

개발자가 프로그램 소스코드를 작성하는 단계이다.
텍스트 편집기를 사용하여 코드 작성이 이루어진다.

예

* Visual Studio Code


② 컴파일(Compile)

작성된 소스코드를 컴퓨터가 이해할 수 있는 기계어 형태의 오브젝트 파일로 변환하는 과정이다.

컴파일러가 사용된다.

예

* GCC


③ 링크(Link)

여러 개의 오브젝트 파일과 라이브러리를 연결하여 최종 실행 파일을 생성하는 과정이다.

이 단계에서 라이브러리가 중요한 역할을 한다.

라이브러리는 프로그램에서 필요한 기능을 제공하며 링크 단계에서 해당 코드가 연결된다.

예

* 수학 라이브러리
* 이미지 처리 라이브러리


④ 실행(Execute)

생성된 실행 파일(.exe)을 실행하여 프로그램이 실제로 동작하는 단계이다.


6. 정적 라이브러리(SLL)와 동적 라이브러리(DLL)

① 정적 라이브러리 (Static Library)

정적 라이브러리는 프로그램 컴파일 시 실행 파일에 직접 포함되는 라이브러리이다.

특징

* 실행 파일에 라이브러리 코드가 포함됨
* 프로그램 크기가 커짐
* 실행 시 별도의 라이브러리가 필요 없음

링크 시점

👉 컴파일 및 링크 단계에서 실행 파일에 포함된다.


② 동적 라이브러리 (DLL)

동적 라이브러리는 프로그램 실행 시 외부에서 로드되는 라이브러리이다.

대표적인 예

* Dynamic Link Library

특징

* 여러 프로그램이 라이브러리 공유 가능
* 실행 파일 크기 작음
* 메모리 효율적

링크 시점

👉 프로그램 실행 시 운영체제가 DLL을 로드한다.


7. DLL 경로가 PATH에 등록되어야 하는 이유

동적 라이브러리(DLL)는 프로그램 실행 시 운영체제가 해당 파일을 찾아서 로드해야 한다.

Microsoft Windows는 DLL을 찾을 때 다음 위치를 검색한다.

1. 실행 파일이 있는 폴더
2. 시스템 폴더
3. 환경변수 PATH에 등록된 폴더

따라서 DLL 파일이 있는 경로를 PATH에 등록하지 않으면 운영체제가 DLL을 찾지 못하여 프로그램이 실행되지 않을 수 있다.

예 오류

DLL file not found

그래서 DLL이 있는 폴더를 PATH 환경변수에 등록해야 프로그램이 정상적으로 실행된다.
